# CLAUDE.md — AI Context File

## What This App Is

**fin451app** — a Golem R package and Shiny dashboard for the University of Alberta FIN451 final project.

The audience is a **senior Risk Management or Trading Management leader** with limited commodity markets experience. The app's job is to tell the story of market dynamics in individual markets and across markets — replacing a Bloomberg terminal with a purpose-built analytical narrative tool.

Everything in the app builds on a single design principle: **data loads once, flows everywhere, and every chart tells a decision-relevant story.**

## How It All Fits Together (Mental Model)

### Layer 1 — Raw Data (the foundation)

Two external data sources, fetched once and cached as Feather binary files in `inst/app/data/`:

**RTL::dflong** — commodity futures prices
- 5 markets: CL (WTI Crude), BRN (Brent Crude), RB (RBOB Gasoline), HO (Heating Oil), NG (Natural Gas)
- Series named by prefix + 2-digit contract number: CL01 = front month, CL02 = second month, etc. Up to CL36.
- Date range: 2007-01-01 to present
- Install: `remotes::install_github("risktoollib/RTL")`

**FRED CMT** — US Treasury Constant Maturity yields
- 11 tenors: DGS1MO, DGS3MO, DGS6MO, DGS1, DGS2, DGS3, DGS5, DGS7, DGS10, DGS20, DGS30
- Requires `FRED_API_KEY` environment variable
- Package: `fredr`

**Caching strategy (`R/fct_data.R`):**
- `fct_fetch_live_data()` — pulls from RTL and FRED, returns a named list
- `cache_app_data()` — calls fetch, writes `futures.feather` and `cmt.feather` to `inst/app/data/`
- Path resolution bug fix: always writes to source tree `inst/app/data/` (checks `dir.exists("inst")` first), not the installed library path
- `fct_load_all_data()` — reads the feather files at startup; derives `futures_front` (wide, M1 prices), `futures_returns` (wide, log returns), `cmt_wide` (wide CMT) in memory

Run once before deployment:
```r
devtools::load_all()
Sys.setenv(FRED_API_KEY = "your_key")
fin451app::cache_app_data()
# commit inst/app/data/futures.feather and cmt.feather to git
```

### Layer 2 — Golem Package Structure

The app is an R package (`fin451app`) built with Golem. This means:
- `R/run_app.R` — entry point: `fin451app::run_app()`
- `R/app_ui.R` — top-level UI
- `R/app_server.R` — top-level server

### Layer 3 — The "Small r" Strategy (how data flows)

This is the core architectural decision. Everything depends on it.

```
app_server.R
  r <- reactiveValues(data = NULL, loaded = FALSE)
  r$data <- fct_load_all_data()          # runs ONCE at startup
  selected_market    <- reactive(input$market)
  selected_daterange <- reactive(input$date_range)

  mod_forward_curve_server("fwd",  r=r, market=selected_market, dates=selected_daterange)
  mod_volatility_server(   "vol",  r=r, market=selected_market, dates=selected_daterange)
  mod_codynamics_server(   "cody", r=r, market=selected_market, dates=selected_daterange)
  mod_seasonality_server(  "seas", r=r, market=selected_market, dates=selected_daterange)
  mod_hedge_ratios_server( "hedge",r=r, market=selected_market, dates=selected_daterange)
  mod_rates_server(        "rates",r=r,                         dates=selected_daterange)
```

- `r$data` is the full dataset — never re-fetched
- `market` and `dates` are reactive expressions passed by reference — when the top-level selector changes, all modules react simultaneously
- Modules filter from `r$data`, never load independently
- `selected_market` and `selected_daterange` are defined at the top level only — no duplicate inputs

The persistent header (market selector + date range) sits in `bslib::page_navbar(header = ...)` so it appears above all tabs without being inside any module. The header is wrapped in `conditionalPanel("input.main_navbar !== 'about'")` so it hides on the Methodology tab. The navbar has `id = "main_navbar"` to enable this.

### Layer 4 — C++ Performance Layer (Rcpp)

`src/roll_vol.cpp` contains two functions, exported via `// [[Rcpp::export]]`:

**`rolling_vol_cpp(x, window)`**
- Input: numeric vector of log returns, integer window
- Algorithm: Welford's one-pass online algorithm for running mean and variance (numerically stable, no catastrophic cancellation)
- Output: rolling annualized vol = running StdDev × √252
- Why C++: ~50× faster than pure R nested loops over 300K+ rows

**`rolling_beta_cpp(x, y, window)`**
- Input: two return vectors + window
- Algorithm: rolling OLS beta = Cov(x,y) / Var(y) computed in one pass
- Output: rolling hedge ratio time series

`R/fin451app-package.R` must contain `@useDynLib fin451app, .registration = TRUE` — without this, `.Call()` to C++ fails at runtime.

### Layer 5 — The Six Analysis Modules

Each module follows the same pattern: **insight card → controls → navset_card_tab with 3 sub-tabs.**

**Forward Curve (`fwd`)**
- What it answers: What shape is the curve? What regime are we in?
- `fct_filter_futures(data, mkt, dates, tenors)` — filters the long futures table
- `fct_roll_yield(data, mkt, dates)` — computes (M1-M2)/M1 × 12, classifies Backwardation/Contango
- Tabs: Structure (snapshot at slider date, optional 4-quarter overlay) | Roll Yield (time series, colored by regime) | Heatmap (tenor × time, price color)
- Export CSV button: downloads all forward curve prices + roll yield for selected market/dates
- Insight card: current regime, annualized roll yield %, historical percentile

**Volatility (`vol`)**
- What it answers: How risky is this market right now vs history?
- Uses `rolling_vol_cpp()` from Rcpp
- Tabs: Time Series (rolling vol with red shading for 90th-percentile spikes, event annotations for GFC/COVID/Russia-Ukraine/US-Iran 2025) | Term Structure (avg vol by contract month — front month always highest) | Surface (heatmap of vol by tenor × time)
- Insight card: current vol percentile rank, peak vol date, regime classification (Low/Normal/High)

**Co-Dynamics (`cody`)**
- What it answers: How do markets move relative to each other?
- IMPORTANT: always shows ALL 5 markets — this is intentional. Selected market determines insight card focus and sets rolling correlation Market 1 default via `updateSelectInput`.
- Info banner in UI explains this to avoid confusion
- Tabs: Correlation Matrix (heatmap, toggleable period and returns/prices) | Rolling Correlations (two-market pair, slider window, auto-updates to selected market) | PCA (scree plot + biplot + auto-interpreted factor labels: PC1 = market level, PC2 = relative value)

**Seasonality (`seas`)**
- What it answers: Are there predictable calendar patterns?
- Tabs: Monthly (bar chart by month; YoY overlay toggle turns it into a spaghetti chart of individual years) | Calendar Heatmap (year × month grid) | Decomposition (STL: trend + seasonal + remainder subplots)
- YoY overlay: `input$seas_overlay` controls whether monthly chart shows aggregated bars or individual-year lines

**Hedge Ratios (`hedge`)**
- What it answers: How many units of Y do I need to hedge 1 unit of X?
- Uses `rolling_beta_cpp()` from Rcpp
- Tabs: Rolling Hedge Ratio (cross-market, slider window) | Term Structure Betas (M1 vs M2-M5, same market) | OLS Summary Table (DT, all cross-market pairs)
- Insight card: current vs long-run beta, drift direction, R²

**Rates / CMT (`rates`)**
- What it answers: What is the yield curve doing and what does it mean for commodities?
- No market selector — always shows all CMT tenors
- Tabs: Yield Curve (snapshot with 1-year-ago overlay) | 10Y Over Time | Term Spread 10Y−3M (red when inverted = recession signal)
- Insight card: curve shape (upward/flat/inverted), 10Y level, spread value

**Methodology (static)**
- `mod_about_ui.R` — pure UI, no server
- Two-column layout of cards explaining: Roll Yield formula, Realized Vol + Rcpp, Hedge Ratio OLS, PCA, STL Decomposition, Data Sources
- Market/date header hidden on this tab via conditionalPanel

### Layer 7 — Deployment (the final layer everything builds toward)

**The problem deployment solves:** the feather cache means no API calls at runtime. The Docker image contains the pre-built package + pre-fetched data. The app starts in <2 seconds with no external dependencies.

**Dockerfile** (in project root):
- Base: `rocker/shiny` or `r-base`
- `COPY . .` — copies source including `inst/app/data/*.feather`
- `RUN R -e "remotes::install_github('risktoollib/RTL')"` — dev version required
- `RUN R -e "devtools::install()"` — builds and installs fin451app with Rcpp compiled
- `EXPOSE 3838`
- `CMD ["R", "-e", "fin451app::run_app(host='0.0.0.0', port=3838)"]`

**GitHub Actions** (`.github/workflows/deploy.yml`):
- Trigger: push to `main`
- Steps: checkout → build Docker image → push to `ghcr.io/<username>/<repo>:latest`
- Required secret: `FRED_API_KEY` (used only if cache needs regeneration at build time)

**Local workflow before pushing:**
```r
# 1. Cache data (source tree)
devtools::load_all()
Sys.setenv(FRED_API_KEY = "your_key")
fin451app::cache_app_data()

# 2. Verify
nrow(arrow::read_feather("inst/app/data/cmt.feather"))     # > 1000
nrow(arrow::read_feather("inst/app/data/futures.feather")) # ~300K

# 3. Install and run
devtools::install()
fin451app::run_app()

# 4. Git
git add inst/app/data/futures.feather inst/app/data/cmt.feather
git commit -m "cache feather data"
git push
# GitHub Actions builds Docker image automatically
```

## Key Design Decisions (and why)

| Decision | Reason |
|----------|--------|
| Feather cache instead of live API | No API timeouts in Docker; sub-2s startup; FRED key not needed at runtime |
| Small-r pattern | One data load, zero duplication, all modules react to same market/date |
| Rcpp for vol + beta | ~50× faster than R over 300K rows; necessary for responsive UI |
| `conditionalPanel` on header | Methodology tab doesn't need market selector; cleaner UX without module refactor |
| `id = "main_navbar"` on page_navbar | Required to read active tab in `conditionalPanel` condition |
| `fct_filter_futures(mkt=)` not `market=` | Avoids NSE variable shadowing — column and arg both named "market" caused silent bugs |
| Co-Dynamics always shows all 5 markets | PCA and correlation are inherently multi-market; selected market sets insight card + rolling corr default |
| Feather written to `inst/` not installed path | `system.file()` resolves to library path when installed; `dir.exists("inst")` check ensures source tree gets the cache |
